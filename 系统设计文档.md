# 客户进件、管理、审批系统 — 轻流搭建设计文档

---

## 一、轻流基础概念

在开始设计之前，先了解轻流平台的三个核心模块：

### 1. 表单（应用）
表单是**数据存储和录入**的载体。轻流中每个"应用"本质上就是一个表单，它定义了：
- 有哪些字段（文本框、下拉框、日期、附件、数字等）
- 字段之间的关联（如选择A类客户后，自动显示A类专属字段）
- 数据的增删改查

可以把它理解为 Excel 表格 + 录入界面的组合。

### 2. 流程
流程定义了**数据如何流转和审批**。节点类型包括：
- **审批节点**：某人/某角色审核，可以同意/驳回/退回
- **填写节点**：指定人员补充信息
- **抄送节点**：通知相关人员，无需操作
- **条件分支**：根据数据不同走不同路径（如金额>10万要总经理批）
- **自动化节点**：自动发消息、更新数据等

### 3. 报表
报表用于**数据分析和展示**：
- 统计图表（柱状图、饼图、折线图）
- 明细表
- 汇总表
- 仪表盘

### 5.3 流程节点详细配置

| 节点 | 操作人 | 可执行操作 | 超时设置 |
|---|---|---|---|
| 客户经理审核 | 动态获取（表单中"客户经理"字段） | 同意/驳回/退回修改 | 24小时提醒 |
| 部门主管审批 | 申请人直属上级 | 同意/驳回/转审 | 48小时提醒 |
| 总监审批 | 指定角色「总监」 | 同意/驳回/转审 | 72小时提醒 |
| 总经理审批 | 指定角色「总经理」 | 同意/驳回/转审 | 72小时提醒 |
| 自动创建档案 | 系统自动 | 创建客户记录、更改状态 | - |
| 消息通知 | 系统自动 | 企微/邮件/站内信通知 | - |

## 六、权限设计

### 6.1 角色定义

| 角色 | 权限范围 | 说明 |
|---|---|---|
| 系统管理员 | 全部数据 + 全部功能 | 配置系统、管理用户 |
| 总经理 | 查看全部客户 + 审批 | 最高审批权限 |
| 总监 | 查看本部门客户 + 审批 | 部门级管理和审批 |
| 部门主管 | 查看本部门客户 + 审批 | 团队管理和审批 |
| 客户经理 | 查看自己负责的客户 + 发起申请 | 一线客户维护 |
| 普通员工 | 仅查看（受限）+ 发起进件 | 最低数据权限 |

### 6.2 数据权限矩阵

| 操作 | 管理员 | 总经理 | 总监 | 部门主管 | 客户经理 | 普通员工 |
|---|---|---|---|---|---|---|
| 查看全部客户 | ✓ | ✓ | - | - | - | - |
| 查看本部门客户 | ✓ | ✓ | ✓ | ✓ | - | - |
| 查看自己客户 | ✓ | ✓ | ✓ | ✓ | ✓ | - |
| 新增客户进件 | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |
| 编辑客户信息 | ✓ | - | - | - | ✓(自己的) | - |
| 审批进件 | ✓ | ✓(金卡+) | ✓(银卡+) | ✓(普通) | - | - |
| 删除客户 | ✓ | - | - | - | - | - |
| 导出数据 | ✓ | ✓ | ✓ | - | - | - |
| 系统配置 | ✓ | - | - | - | - | - |

### 6.3 轻流权限配置要点

1. **表单权限**：在"表单设置 → 权限管理"中配置
2. **流程权限**：通过审批节点中的"操作人"配置
3. **报表权限**：在仪表盘中设置可见范围
4. **数据权限**：使用"数据权限规则"按部门/角色过滤

## 八、轻流搭建步骤指南

### 第一阶段：基础配置（1-2天）

**步骤1：创建工作区**
1. 登录轻流管理后台
2. 新建工作区「客户管理系统」
3. 邀请团队成员加入

**步骤2：搭建客户信息表（普通表单）**
1. 进入「应用管理」→「新建应用」→「普通表单」
2. 命名为「客户信息表」
3. 按照3.1节字段表逐个添加字段
4. 配置「客户编号」为流水号（前缀：KH-{YYYYMMDD}-）
5. 在「表单设置」中配置字段显隐规则
6. 保存并发布

**步骤3：搭建进件申请表（流程表单）**
1. 新建应用 →「流程表单」
2. 命名为「客户进件申请」
3. 添加字段（参考4.1节）
4. 添加「关联数据」字段，关联到客户信息表
5. 配置显隐规则（企业客户字段组）
6. **重要**：流程表单先不要配置流程，先保存表单结构

### 第二阶段：流程配置（1-2天）

**步骤4：配置进件审批流程**
1. 在进件申请表中进入「流程设计」
2. 拖入节点，按照5.1节流程图搭建
3. 配置条件分支：
   - 条件1：客户等级 = 普通/银卡 → 走短路径
   - 条件2：客户等级 = 金卡/钻石 → 走长路径
4. 配置每个审批节点的操作人
5. 配置自动节点：
   - 审批通过 → 自动在客户信息表创建记录
   - 审批通过 → 发送通知
   - 审批驳回 → 发送通知
6. 设置超时提醒
7. 测试流程

**步骤5：配置变更审批流程**
1. 新建流程表单「客户变更申请」
2. 添加关联客户信息表的字段
3. 按照5.2节配置变更审批流程
4. 自动节点：审批通过后更新客户信息表

### 第三阶段：权限配置（0.5天）

**步骤6：角色和权限配置**
1. 在「系统管理」→「角色管理」创建角色
2. 按照6.1节创建6个角色
3. 为每个表单配置权限（表单设置 → 权限管理）
4. 配置数据权限规则（按部门/客户经理过滤）

### 第四阶段：报表搭建（1天）

**步骤7：创建报表**
1. 进入「报表中心」→「新建报表」
2. 创建7.2节列出的报表
3. 拖入图表组件，配置数据源和维度
4. 组装仪表盘
5. 配置权限（谁可以看哪些报表）

### 第五阶段：测试和上线（1天）

**步骤8：全流程测试**
1. 用测试账号走一遍完整流程
2. 测试各角色权限是否生效
3. 测试审批驳回/转审等分支场景
4. 测试超时提醒
5. 修复问题

**步骤9：上线**
1. 导入正式客户数据（如有存量）
2. 通知全员培训
3. 正式切换

## 九、关键注意事项

1. **关联数据**：进件表和客户表之间用「关联数据」字段连接，审批通过后自动回写
2. **版本管理**：轻流支持表单发布历史，改配置前先保存版本
3. **数据备份**：定期从报表中导出客户数据做离线备份
4. **审批人动态获取**：用轻流的组织架构对接能力，不要写死审批人
5. **移动端适配**：轻流自动适配移动端，表单字段不宜过多，用分组折叠
6. **附件存储**：注意轻流的附件存储空间上限，大文件建议外链
